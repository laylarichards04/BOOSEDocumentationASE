Namespace BOOSEAppASE

Classes

AppArray

Allows integer and real arrays to be declared without restrictions by overriding the BOOSE array command. These arrays are stored in AppVariableStore as AppArrayVariables.

AppArrayVariables

Represents a runtime array variable that supports both integer and real arrays. All values are stored as doubles, as integers automatically round values. Array, poke and peek all use this class.

AppCall

Executes a previously defined method. Argument values are then passed into method parameters. A new local scope is pushed to AppVariableStore so method parameters act as local variables.

AppCanvas

A class creating a canvas interface for BOOSE, implementing BOOSE.ICanvas. Provides a number of methods for drawing/editing including Circle(int, bool), Rect(int, int, bool), MoveTo(int, int) and DrawTo(int, int).

AppCircle

A class to create the updated Circle command, referenced in the AppCommandFactory. The class overrides BOOSE limitations, allowing a circle of any size to be drawn on the canvas.

AppCommandFactory

A command factory that overrides or extends the original BOOSE BOOSE.CommandFactory. Commands are created in this class to overcome limitations in the base Command Factory.

AppConditions

Evaluates BOOSE conditional expressions for commands like if, while and for. AppExpressions evaluates both sides of the expression so that ints and reals can both be compared.

AppEvaluation

Implements runtime assignment evaluation for expressions. The left variable is stored in BOOSE.Evaluation.VarName. The right variable is stored in BOOSE.Evaluation.Expression.

AppExpressions

Parses and evaluates expressions.

AppIf

Represents an unrestricted if command in BOOSE. The command evaluates to either true or false based on the condition, then the correct code block is executed. The condition is evaluated using AppConditions.

AppInt

Represents the unrestricted int declaration command in BOOSE. Handles both integers and expressions. Expressions are handled by AppExpressions.

AppMethodDefinition

Represents a user-created method in BOOSE. Used when a method is called in the above 'Get'.

AppMethodStore

Central storage for user-created methods in BOOSE. Implements singleton design pattern as all method definitions are kept in a single, shared location.

AppMoveTo

A class to create the updated Moveto command, referenced in the AppCommandFactory. The class overrides BOOSE limitations, allowing the pen to be moved to any point on the canvas with x and y coordinate values.

AppParser

Custom implementation of the BOOSE parser. Converts BOOSE code into executable command objects.

AppPeek

Implements the peek command that reads an array value and assigns it to a variable.

AppPen

Implements the BOOSE pen command, allowing the pen colour to be set using RGB values. This supports both integers and variable names for the RGB values.

AppPoke

Implements the poke command, used to write a value into an array at a given index. The value may be a number or expression.

AppReal

Implements the unrestricted real command, allowing the declaration of a real variable.

AppRect

A class to create the updated Rectangle command, referenced in the AppCommandFactory. The class overrides BOOSE limitations, allowing a rectangle of any width and height to be drawn on the canvas.

AppValueResolver

Helper methods for resolving values from tokens. This supports literals, variables or expressions. Centralises the code for value resolution, making it easy for commands to accept integers, expressions and variables easily.

AppVariableStore

A store that manages variables for BOOSE, using a singleton design pattern. It is the single variable store that is shared across the entire program.

AppWhile

Represents an unrestricted while loop BOOSE command. AppConditions is used to evaluate the condition strings.

AppWriteText

Implements the BOOSE write command to write text to the canvas, starting at the pen position. This class can handle expressions and concatenate strings.

Form1

The primary User Interface for drawing with BOOSE, extending Form. Creates a physical interface with a command input box, drawing canvas box, run button and an error output box.

TraceCanvas

A decorator design pattern implementation of ICanvas that records a trace stack of all drawing commands drawn to the canvas. Wraps existing ICanvas instance and intercepts method calls to record and display them as trace output. Execution is still done by the original canvas.

Interfaces

IVariableStore

Defines a contract/rules for the singleton variable storage. Responsible for declaring variables, updating variable values, retrieving variable values, checking variable existence and clearing stored variables. AppVariableStore provides an implementation of this interface using a stack of dictionaries.
